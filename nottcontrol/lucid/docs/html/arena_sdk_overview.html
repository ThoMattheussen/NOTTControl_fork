Arena SDK: Overview

Arena SDK: Overview#

Introduction#

The purpose of the Arena SDK is to allow custom applications to enumerate, configure, and trigger LUCID cameras, as well as to obtain image data.

The Arena SDK was created with simplicity and ease of use in mind. Users of existing GeniCam cameras will be able to quickly understand the SDK, and the code samples provided may help new users create custom applications.

The Arena SDK has APIs for C++, C, .NET (C#), and Python that let you access and configure all available camera features from within your application.

Related standards#

The Arena SDK implements the following standards:

  • GenICam standard (GenAPI): Defines how high-level requests from applications are translated into specific commands that are compatible with the camera’s register-level interface.

  • GenICam GenTL standard: Defines the transport layer interface, which specifies how modules should be implemented for the following layers: System, Interface, Device, Stream, and Buffer. See the C++ Programmer’s Guide for an example of how these modules are implemented by LUCID.

Arena SDK architecture#

Applications written in the various available programming languages can access camera functions by accessing the applicable API.

Image packets from camera are transported via Ethernet to the network interface card, where they are passed up the network stack to the API and then to the application. Different protocols are used to handle control packets and streaming packets for improved speed.

The base API for the Arena SDK is written in C++, and the APIs for the other programming languages are built on top of this base API.

Separately from this data flow, the GenAPI portion of the SDK also reads the nodes of the XML file in the camera, which lets upstream applications read and configure settings related to camera features.

The Windows and Linux implementations differ in the details of the network stack; in addition, C# is not supported in the Linux version of the SDK.

Windows#

Apps & third-party software#

  • Apps: Apps written in Python, C, C++, and C# can access the SDK through various Arena APIs.

  • Third-party software: Developers of third-party software can write a GenTL consumer component that lets the software directly access the GenTL interface for the camera.

Arena#

  • APIs: The Arena SDK has Python, C, and C++ APIs, which allows applications written in each of these languages to use the SDK. In Windows, the Arena SDK also has a .NET API with C# code samples.

  • GenICam GenAPI (C++ API): The GeniCam GenAPI implementation is part of the C++ API, and provides a method of configuring a camera in a generic way as specified in the GenICam standard.

  • GenTL producer interface: A software driver that implements the GenTL interface as defined in the GenICam GenTL standard. This allows the transport layer of the camera to be accessed in a generic way by the C++ API and by GenTL consumers. GenTL consumers for third-party software access GenTL producers with a CTI file extension.

  • GVCP: GigE Vision Control Protocol. Control packets, which describe how devices are controlled and configured, are handled following this protocol.

  • GVSP: GigE Vision Stream Protocol. Image data packets are handled following this protocol. Note: For Atlas10 cameras streaming in TCP, GVSP packets are sent via TCP.

Network stack#

  • Lightweight Filter Driver: A low-level filter driver that checks for missing packets and sends out packet resend requests.

  • NIC and NIC driver: Network interface card and driver.

Camera#

  • Camera: Manufactured by LUCID Vision Labs.

  • Register: The low-level configuration tables found on the camera. The XML file translates nodes to register addresses.

  • XML file (camera): A file format defined in the GenICam standard that allows access to camera controls and image streams through the main node map. The nodes of the main node map obtained from the device in the SDK correspond to the nodes of the XML file.

Linux#

Apps & third-party software#

  • Apps: Apps written in Python, C, and C++ can access the SDK through various Arena APIs.

  • Third-party software: Developers of third-party software can write a GenTL consumer component that lets the software directly access the GenTL interface for the camera.

Arena#

  • APIs: The Arena SDK for Linux has Python, C, and C++ APIs, which allows applications written in each of these languages to use the SDK.

  • GenICam GenAPI (C++ API): The GeniCam GenAPI implementation is part of the C++ API.

    It provides a method of configuring a camera in a generic way as specified in the GenICam standard.

  • GenTL producer interface: A software driver that implements the GenTL interface as defined in the GenICam GenTL standard. This allows the transport layer of the camera to be accessed in a generic way by the C++ API and by GenTL consumers. GenTL consumers for third-party software access GenTL producers with a CTI file extension.

  • GVCP: GigE Vision Control Protocol. Control packets, which describe how devices are controlled and configured, are handled following this protocol.

  • GVSP: GigE Vision Stream Protocol. Image data packets are handled following this protocol. Note: For Atlas10 cameras streaming in TCP, GVSP packets are sent via TCP.

Network stack#

  • BSD Sockets interface: A standard API for inter-process communication used in Linux machines.

  • NIC and NIC driver: Network interface card and driver.

Camera#

  • Camera: Manufactured by LUCID Vision Labs.

  • Register: The low-level configuration tables found on the camera. The XML file translates nodes to register addresses.

  • XML file (camera): A file format defined in the GenICam standard that allows access to camera controls and image streams through the main node map. The nodes of the main node map obtained from the device in the SDK correspond to the nodes of the XML file.

Interfaces#

The Arena SDK interfaces in many ways parallel the GenTL layers, and includes interfaces to the System object, Device object, and Buffer object. The System object is the entry point into the Arena SDK; this object must be instantiated before anything else. Not all GenTL layers are reflected as Arena SDK objects because they have been abstracted away for ease of use. For instance, no Interface object is needed in the Arena SDK because LUCID cameras are all GigE Vision cameras.

See the Code Reference and Samples for more details on Arena SDK classes.

Ports#

In general, you don’t have to explicitly open ports under Windows when using the built-in firewall. Windows automatically allows incoming traffic on ports where communication is initiated by the host.

For Linux systems, you may need to unblock ports if they are being blocked by a system firewall.

The Microsoft Visual C++ Redistributables should be installed first. The redistributable to be installed depends on the architecture of the program.

Distributing Arena SDK Programs for Linux#

This section describes the requirements for distributing your Arena SDK-based program to another Linux system.

x64 Linux#

The following folders are found in the Arena SDK tarball for x64 Linux. Ensure that their contents are available to your application.

  • /lib64

  • /GenICam/library/lib/Linux64_x64

  • /ffmpeg

In the Linux64_x64 folder, there are several versions of the .so files. No suffix means it is for C++, a “c” suffix means that it is for C. A “d” suffix indicates that it is a debug library.

ARM Linux#

The following folders are found in the Arena SDK tarball for ARM Linux. Ensure that their contents are available to your application.

  • /lib

  • /GenICam/library/lib/Linux64_ARM

  • /ffmpeg
